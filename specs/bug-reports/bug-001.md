# Bug Report 001

## Title
Salary deduction shows wrong total when
potongan value is zero

## Severity
High

## Who Gets Hurt
Payroll operator processes wrong salary.
Worker gets paid wrong amount and does not
know until next month.

## Steps to Reproduce
1. Login as Admin
2. Go to Setting Potongan Gaji
3. Set Sakit deduction to Rp. 0
4. Go to Data Gaji
5. Check Total Gaji column

## Expected Result
Total Gaji = Gaji Pokok + Tunjangan Transport
+ Uang Makan - 0 (no deduction)

## Actual Result
Total calculation behaves unexpectedly when
deduction value is zero instead of null

## Root Cause
Zero value and null/empty are treated
differently in salary calculation logic

## Impact
If deduction is accidentally set to zero,
every worker salary total may be wrong
for that month without any warning shown
