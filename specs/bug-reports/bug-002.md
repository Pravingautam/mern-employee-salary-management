# Bug Report 002

## Title
Employee form accepts negative salary value
without any error or warning

## Severity
Critical

## Who Gets Hurt
If admin accidentally enters negative salary,
worker gets zero or wrong payslip that month.
Payroll operator has no way to know until
salary report is printed.

## Steps to Reproduce
1. Login as Admin
2. Go to Data Jabatan
3. Click Tambah Jabatan
4. Enter Gaji Pokok as -1000000
5. Click Simpan

## Expected Result
System should reject negative salary value
and show error message

## Actual Result
Form accepts negative value and saves it
No validation error shown

## Root Cause
Frontend form has no minimum value check
on salary input field

## Impact
Worker salary calculation will produce
wrong total. This is a critical payroll
data integrity issue.
